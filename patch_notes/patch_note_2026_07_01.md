# WE-MEET 패치노트 (2026.07.01 - 긴급 장애 해결 및 GUI 고도화 종합 패치)

본 패치노트는 이기종 가상 클러스터 분산 학습 런타임인 **WE-MEET**의 스팟 인스턴스 이기종 확장 연동 과정에서 발생한 시스템 크래시, 데이터셋 네트워크 차단, 맵-리듀스 데드락, 셧다운 좀비 누수 장애 해결 및 프리미엄 대시보드 UI 고도화 종합 패치 내역을 기술합니다.

---

## 🛠️ 1. 긴급 빌드 및 런타임 크래시(NameError/ImportError) 해결
* **`head/cluster_manager.py` 내 필수 표준 라이브러리 임포트 보완**:
  * **장애 증상:** 헤드 노드가 동적 기동하거나 스팟 강제 회수 데몬이 실행될 때 `NameError: name 'threading' is not defined`, `name 'time' is not defined`, `name 'random' is not defined` 에러가 나며 컨테이너가 즉시 다운되는 크래시가 발생했습니다.
  * **해결 내용:** 파일 상단에 누락되어 있던 `import threading`, `import time`, `import random` 구문을 완벽히 주입했습니다.
* **`worker/gpu_simulator.py` 의 불필요한 레거시 임포트 제거**:
  * **장애 증상:** 리팩토링으로 다형성 캡슐화되어 삭제된 레거시 함수 `train_one_epoch`와 `run_model_inference`를 파일 상단에서 계속 가져오려 시도하여 `ImportError`가 발생해 워커 컨테이너가 켜지자마자 죽던 현상을 해결했습니다.

---

## 📥 2. MNIST 데이터셋 외부 네트워크 404 차단 이슈 해결
* **인라인 로컬 MNIST 가상 픽셀 데이터셋 구현 (`cnn.py`)**:
  * **장애 증상:** `torchvision.datasets.MNIST`가 내부적으로 사용하는 AWS S3 기본 다운로드 URL 경로 만료 또는 일시 서버 다운으로 인해 `HTTP Error 404: Not Found`가 발생하며 분산 훈련 연산이 지연되거나 중단되는 현상이 있었습니다.
  * **해결 내용:** 외부 다운로드 시도를 100% 제거하고, 숫자 0~9의 고유한 뼈대 모양을 28x28 픽셀 형태의 다차원 텐서로 코드 내부에 그린 **`get_inline_mnist_dataset()` 인라인 데이터셋** 방식을 전면 적용했습니다. 인터넷 차단 및 404 에러와 무관하게 0초 만에 훈련이 완료되며, 추론 시 실제 숫자 3을 정확히 분류해 내는 유의미한 예측 정확도가 보장됩니다.

## 🔒 3. Map-Reduce 분산 가중치 병합(Reduce) 실패 및 데드락 해제
* **도커 공유 네임드 볼륨(`babyray-data`) 도입**:
  * **장애 증상:** 다중 컨테이너 환경의 격리된 파일시스템으로 인해 각 워커가 저장한 맵 연산 가중치 파일(`.pt`)을 병합 워커가 읽지 못해 `FileNotFoundError`와 함께 병합 단계가 무조건 실패하던 문제를 해결했습니다. `docker-compose.yml` 및 컨테이너 런타임에 동일 공유 스토리지 영역을 바인딩했습니다.
* **병합 작업 온디맨드 노드 전용 고정 (Reduce Pinning)**:
  * **장애 증상:** 가중치 결합 도중 전담 스팟 노드가 Eviction(회수)으로 날아가 `Reduce worker went offline` 에러를 뿜고 유실을 겪던 현상을 방지하기 위해, 병합 작업만큼은 탈락 위험이 전혀 없는 안전한 `on_demand` 워커로 고정시켰습니다.
* **스케줄러 순환 대기 데드락(Deadlock) 해제**:
  * **장애 증상:** 주체 온디맨드 워커가 분산 연산을 주도할 때 맵 작업이 끝나 `IDLE`로 릴리즈되었어야 하나, 메인 스레드 점유로 인해 `BUSY` 락이 풀리지 못해 병합 단계에서 영원히 순환 대기하던 데드락 현상을 맵-리듀스 진입 시 락을 해제하도록 보완하여 해결했습니다.

## 🛡️ 4. 시스템 메모리 가드 임계치 상한 및 예외 우회 장치 마련
* **메모리 부족 안전 가드 임계치 `4.0 GB` 상향 및 가이드 강화**:
  * 개발 장치의 가용 메모리가 부족한 상태에서 강제로 컨테이너를 적재하면 OS 붕괴가 우려되므로, 검사 임계 안전선을 `4.0 GB`로 재설정하고 부족 시 WSL2/도커 데스크톱의 할당 메모리를 4GB 이상으로 증가시키라는 상세 해결책 로그를 출력하도록 개선했습니다.
* **`BYPASS_RESOURCE_GUARD` 환경 변수 신설**:
  * 로컬 개발/실습 편의성을 위해, `$env:BYPASS_RESOURCE_GUARD="1"` 환경 변수 주입 시 호스트 램 자원 부족 검사를 강제로 스킵하고 스팟 노드를 즉시 팽창시킬 수 있는 우회 옵션을 연동했습니다.

## 🔌 5. Graceful Shutdown (좀비 누수 소거) 및 네트워크 릭 방지
* **시그널 처리 종료 훅 (`SIGINT/SIGTERM`) 구현**:
  * **장애 증상:** `Ctrl+C` 또는 `docker-compose down`으로 끌 때 동적으로 기동된 스팟 워커 컨테이너들이 정리되지 않고 호스트에 누수되어 네트워크 포트를 점유하고 있던 문제를 해결했습니다. 헤드 노드가 꺼질 때 살아있던 동적 스팟 워커들을 도커 SDK로 동기 청소하도록 셧다운 훅을 적용했습니다.
* **외부 네트워크 (`external: true`) 구성**:
  * `docker-compose down` 시 컨테이너 종료 지연으로 인해 발생하던 `Network Resource is still in use` 에러를 방지하고, 재실행 시 네트워크 꼬임 현상을 방지하기 위해 `babyray-net` 브릿지 네트워크를 상시 영속되는 외부 네트워크 구조로 개편했습니다.
